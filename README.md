# p1-w2-d1-t1
# Задача 1 — habit: трекер привычек в терминале

Первая серверная утилита. Никакого браузера — программа запускается командой `node`, читает аргументы командной строки и печатает ответ в терминал. Данные пока держим в памяти процесса (при перезапуске обнуляются — так и задумано на этом шаге).

## Подготовка проекта

Собери новый проект по шагам с лекции:

1. Папка и git: `mkdir habit && cd habit && git init`
2. Зафиксируй версию Node: `echo "lts/*" > .nvmrc && nvm use`
3. `npm init -y`
4. `.gitignore` под Node: минимум `node_modules`, `.env`, `*.log`
5. Поставь Prettier и ESLint: `npm install -D prettier eslint`, затем `npx eslint --init`
6. `.prettierrc` с правилами проекта (`singleQuote`, `semi`, `printWidth`) и `.prettierignore`
7. Точка входа `habit.js`, в `package.json` добавь скрипт `"start": "node habit.js"`

В `package.json` тип модулей — `commonjs`. Экспорт и импорт делаешь через `module.exports` / `require`.

## Что за утилита

`habit` — счётчик отметок по привычкам за день. Пользователь вызывает программу с командой и аргументами, программа отвечает строкой в терминал. Работать должно так (это примеры запусков, не готовый код):

```
node habit.js add "чтение"
node habit.js done "чтение"
node habit.js done "чтение"
node habit.js list
node habit.js top
```

Список привычек и счётчик отметок держишь в обычном массиве/объекте внутри процесса.

## Команды

Программа разбирает `process.argv` и по первому смысловому аргументу выбирает команду:

- `add <название>` — завести новую привычку с нулём отметок. Если привычка с таким названием уже есть — сказать об этом, второй раз не заводить.
- `done <название>` — добавить одну отметку существующей привычке. Если такой привычки нет — сообщить, что сначала её надо завести.
- `list` — показать все привычки со счётчиками, по одной в строке.
- `top` — показать привычку с наибольшим числом отметок. При равенстве — любую из лидеров.
- команда не распознана или отсутствует — показать короткую справку со списком доступных команд.

## Входные данные

Стартовый набор привычек, с которого программа поднимается (захардкодь его в `habit.js` как начальное состояние, чтобы было что показывать сразу):

```js
const initial = [
  { name: 'чтение', count: 3 },
  { name: 'зарядка', count: 5 },
  { name: 'английский', count: 1 },
];
```

## Разбей на модули

- `parseArgs.js` — принимает `process.argv`, возвращает объект вида `{ command, argument }`. Служебные первые два элемента argv сюда не тянутся. Экспортируй наружу через `module.exports`.
- `store.js` — состояние (массив привычек) и функции работы с ним: добавить, отметить, вернуть список, вернуть лидера. Ни одной строки про `process` и про печать — только данные.
- `habit.js` — точка входа: берёт разобранные аргументы из `parseArgs`, дёргает нужную функцию из `store`, печатает результат в терминал.

## Ожидаемый результат

На стартовом наборе `node habit.js list` выводит три привычки со счётчиками 3, 5 и 1 — каждую на своей строке. Команда `top` на этом же наборе называет «зарядка» как привычку с наибольшим числом отметок. После `add "вода"` в списке появляется четвёртая привычка с нулём; повторный `add "вода"` состояние не меняет и печатает, что такая уже есть. После двух `done "чтение"` её счётчик становится 5. Вызов `done "медитация"` (такой привычки нет) ничего не заводит и объясняет, что привычку сначала надо добавить. Запуск без команды или с незнакомой командой печатает справку. Логика разбора аргументов, хранения и печати лежит в трёх разных файлах и общается только через экспортируемые функции.
